# CASPBX

[![License](https://img.shields.io/github/license/casapps/caspbx)](LICENSE.md)

## About

`caspbx` is being built as a unified Asterisk communications platform that replaces the need for multiple separate PBX, operator, fax, end-user, call-center, and backend administration interfaces.

The repository is currently in the **bootstrap and architecture phase**. The product definition is extensive and research-backed, and the codebase now contains the initial Go module, build scaffolding, documentation skeleton, and test harness needed to grow into the full platform.

## Features

- Research-backed product definition for a full Asterisk communications platform
- Minimum feature/capability parity target for the replacement surfaces
- Built-in, no-plugin product model with no feature gating
- Capability-driven UI policy so unusable features stay hidden until available
- Initial Go, Makefile, documentation, and container-testing bootstrap

## Production

Production deployment targets for `caspbx` are planned to include:

1. Container deployment under `docker/`
2. Native binary deployment for the `caspbx` server binary
3. Service-manager installation flows for long-running server operation

The runtime server implementation is not complete yet, so this repository should currently be treated as a **foundation and specification bootstrap**, not a finished deployable PBX.

## Client

The project spec requires a companion `caspbx-cli` client and an optional `caspbx-agent`, but those binaries are not implemented yet in this bootstrap stage.

## Configuration

Configuration files are runtime-generated and are **not committed** to the repository.

Current bootstrap status:

- `server.yml` is not committed and will be generated by the server on first run
- project paths, Docker layout, and documentation layout are now scaffolded
- the full runtime configuration model will follow the spec as server code lands

## API

The final product will expose server, admin, and operational APIs, but the API implementation is not present yet.

Current bootstrap behavior:

- `src/` contains a minimal bootstrap binary scaffold
- `docs/api.md` currently documents the intended documentation structure rather than a live API surface

## Other

### Current repository status

This repository currently contains:

- the source-of-truth project specification in `AI.md`
- a comprehensive `IDEA.md` focused on the product **WHAT**
- rule files and project memory files
- bootstrap source, documentation, test, and build scaffolding

### Planned product scope

The target platform replaces the practical need for multiple separate surfaces, including:

- PBX administration
- end-user communications portal
- operator and receptionist switchboard
- fax administration and user workflows
- call-center reporting and live supervision
- system and backend operational administration
