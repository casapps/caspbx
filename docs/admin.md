# Admin Panel

## Current Status

The admin panel is not implemented yet. This document currently records the intended administrative surface so the documentation can evolve alongside implementation.

## Planned Admin Surface

- `/admin` as the primary administrative path
- `/admin/server/asterisk` for backend-stack administration
- role-aware access and secure defaults
- capability-driven UI visibility
- no routine terminal dependence after installation

## Product Direction

The goal is a single coherent admin experience that replaces fragmented PBX, fax, operator, reporting, and backend administration surfaces around Asterisk.
